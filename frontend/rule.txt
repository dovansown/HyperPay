RULE FRONTEND HYPERPAY

1) Scope page public
- chi lay content (main), khong copy sidebar/header/footer tu design.
- header/footer dung component dung chung.

2) i18n
- toan bo text hien thi tren UI phai qua i18n.
- khong hardcode chu tren button/label/status neu da co key.
- text tam (debug/internal) duoc phep hardcode nhung phai toi gian.

3) Router
- moi page moi phai khai bao route trong src/router/router.tsx.
- route can auth phai boc withAuth.
- route can role phai boc withRole (AUTHOR/EDITOR/ADMIN cho CMS).

4) API integration
- dung apiFetch trong src/lib/apiClient.ts.
- uu tien unwrapApiData cho response envelope { success, data, ... }.
- khong parse response theo kieu "doan format" tai tung page.
- loi API phai hien thi thong diep ro rang o UI.

5) State management
- du lieu dung chung phai dua vao Redux slice (store/*Slice.ts).
- page chi local state cho filter/form UI.
- co loading/error state cho moi call bat dong bo.

6) Mapping backend schema
- Webhook phai map day du field theo be/schema + webhooks.schema.ts.
- Transactions phai map dung truong occurred_at/type/payment_code/balance.
- Billing phai dung plans/packages/me-active, khong dung mock data.
- Content public phai dung /public/content (blog/docs/help).

7) UX minimum
- moi list page phai co: loading, empty, error.
- uu tien co filter co ban (search/type/date hoac status).
- button action chinh phai disable/loading khi dang submit.

8) Quality gate
- sau thay doi lon phai chay npm run build (frontend/front).
- khong merge code neu build fail.
- giu code style dong nhat, ten bien ro nghia.
